# Immediate Execution Checklist (Start Now)

## 0-24 Hours: Command Setup
- Confirm CEO sponsor and CTO sponsor for this program.
- Assign owners for 5 squads: Student, Teacher, Institution, AI/Data, Growth.
- Freeze MVP scope for 90 days (no net-new scope without approval).
- Publish this execution pack to all leads.

## 24-72 Hours: Delivery Operating System
- Start daily standup and weekly release cadence.
- Create single source roadmap board with phase gates.
- Approve KPI dictionary and event tracking schema.
- Lock design system v1 foundation tokens.

## Week 1: Product and Design
- Finalize student dashboard UX to approved reference style.
- Finalize journey, explore, and profile consistency pass.
- Complete design QA checklist and accessibility baseline.
- Prepare usability test scripts for student pilot group.

## Week 1: Engineering and Data
- Implement production API contract for dashboard feeds.
- Implement analytics events for all P0 student actions.
- Set up monitoring: uptime, latency, client errors.
- Create learner profile schema and progress endpoints.

## Week 2: Pilot Readiness
- Select and onboard 5-10 pilot institutions.
- Configure role access for students, teachers, admins.
- Run security/privacy checklist and close critical gaps.
- Prepare pilot support playbook and escalation channel.

## Week 3-4: Outcome Tracking
- Launch pilot cohort.
- Track baseline KPIs daily.
- Publish weekly executive report with decisions/actions.
- Prioritize fixes by impact on learning outcomes.

## Executive Review Rituals
- Monday: delivery and blocker review.
- Wednesday: product and pedagogy deep dive.
- Friday: KPI review and decision log.

## Red-Line Escalation Rules
- Any P0 security issue -> immediate release freeze.
- Any >5% drop in core engagement KPI -> 24h intervention plan.
- Any AI safety incident -> immediate policy and model review.
